# Завдання 6. Жадібні алгоритми та динамічне програмування

## Опис

У цьому завданні реалізовано два підходи для вибору їжі в межах заданого бюджету:

- жадібний алгоритм;
- алгоритм динамічного програмування.

Мета обох підходів — вибрати страви так, щоб отримати найбільшу сумарну калорійність і не перевищити бюджет.

## Реалізація

Дані про страви зберігаються у словнику `items`. Для кожної страви задано її вартість та калорійність.

Функція `greedy_algorithm` сортує страви за співвідношенням калорій до вартості. Після цього вона послідовно додає страви з найкращим співвідношенням, якщо вони не перевищують доступний бюджет.

Функція `dynamic_programming` використовує таблицю, де рядки відповідають стравам, а стовпці — можливим значенням бюджету. У кожній комірці зберігається максимальна кількість калорій, яку можна отримати для відповідного бюджету.

Після заповнення таблиці програма відновлює набір страв, який дає найкращий результат.

## Результат

Для бюджету `100` жадібний алгоритм вибирає:

- `cola`;
- `potato`;
- `pepsi`;
- `hot-dog`.

Загальна вартість: `80`.

Загальна калорійність: `870`.

Алгоритм динамічного програмування вибирає:

- `pizza`;
- `pepsi`;
- `cola`;
- `potato`.

Загальна вартість: `100`.

Загальна калорійність: `970`.

У цьому прикладі динамічне програмування знаходить кращий результат, тому що перевіряє різні комбінації страв у межах бюджету.

Основний файл рішення: `main.py`.
