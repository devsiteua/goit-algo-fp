# Завдання 7. Метод Монте-Карло

## Опис

У цьому завданні реалізовано моделювання кидання двох гральних кубиків методом Монте-Карло.

Програма виконує велику кількість кидків, обчислює суму значень двох кубиків, підраховує кількість появ кожної можливої суми від 2 до 12 та визначає ймовірність для кожної суми.

Після цього отримані результати порівнюються з аналітичними ймовірностями.

## Реалізація

Функція `roll_dice` моделює один кидок двох кубиків і повертає суму значень.

Функція `monte_carlo_simulation` виконує задану кількість кидків і підраховує, скільки разів випала кожна сума від 2 до 12.

Функція `calculate_probabilities` переводить кількість появ кожної суми у відсотки.

Функція `get_analytical_probabilities` створює таблицю аналітичних ймовірностей. Вона базується на тому, що для двох кубиків існує 36 можливих комбінацій.

Для повторюваного результату в основному прикладі використано `random.seed(42)`.

## Висновки

Результати, отримані методом Монте-Карло, дуже близькі до аналітичних значень.

Найбільша ймовірність отримана для суми 7. Це відповідає аналітичному розрахунку, тому що суму 7 можна отримати найбільшою кількістю комбінацій: 6 з 36.

Найменші ймовірності отримані для сум 2 і 12. Це також відповідає аналітичним розрахункам, тому що кожну з цих сум можна отримати тільки однією комбінацією.

Невеликі відхилення між результатами симуляції та аналітичними значеннями є нормальними, тому що метод Монте-Карло використовує випадкові експерименти. При великій кількості кидків результати наближаються до теоретичних ймовірностей.
